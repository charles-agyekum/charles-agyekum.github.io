SELECT r.region, COUNT(*) AS orders,
  SUM(CASE WHEN o.late = 1 THEN 1 ELSE 0 END) AS late_orders
FROM orders o
JOIN regions r ON o.region_id = r.id
WHERE r.id IN (SELECT id FROM regions WHERE active = 1)
GROUP BY r.region HAVING late_orders > 0
ORDER BY late_orders DESC;

What this is

A worked set of 31 SQL questions covering the core of analytical querying — joins (inner, left and the rest), subqueries, and CASE logic for banding and recoding. The query files and a reference cheat sheet are kept together so the working is visible, not just the answers.

Coverage

  1. 1 Joins — across related tables, including left joins and the cases where rows go missing if you pick the wrong one.
  2. 2 Subqueries — for filtering and for derived values.
  3. 3 CASE — banding continuous values and recoding categories.
  4. 4 Cheat sheet — kept as a quick reference alongside the query files.

How it connects

This is the grounding under the dashboards. The 1/0-flag-to-rate move in the HR Attrition piece is the spreadsheet version of AVG(flag); the measures in the Power BI builds are the same aggregation logic in DAX. SQL is where that thinking was drilled first.
